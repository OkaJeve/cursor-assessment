# Developer Workspace Setup & Expert Data Collection Pipeline

This repository documents the step-by-step terminal environment configuration for modern AI-assisted engineering workflows, followed by a hyper-targeted data collection sprint tracking 10 industry-leading SEO, AEO, and operational execution practitioners.

---

## 🛠️ Tools Installed
* **Cursor IDE**: Advanced AI-first code editor.
* **Claude Code for VS Code** by Anthropic: Deeply integrated context agent for inline terminal execution.
* **Codex - OpenAI's Coding Agent**: Agentic assistant for code automation tasks.
* **Git for Windows**: Version control interface for remote synchronization.

---

## 🚀 Steps Completed

### Phase 1: Environment Configuration
1. Downloaded and initialized the core **Cursor IDE** package.
2. Deployed **Claude Code** and **Codex** extensions directly through the integrated developer marketplace.
3. Created and mapped a public remote tracking branch on GitHub.
4. Installed and configured **Git for Windows** global user parameters.
5. Synced local operational folders with the remote main branch inside the Cursor terminal.

### Phase 2: Expert Data Sourcing & Architecture Pipeline
We curated, filtered, and structured tactical insights from 10 verified industry practitioners who actively execute what they teach. Using custom API tooling (including the `Supadata` transcript engine), we built a clean, machine-readable strategy layer completely stripped of social media fluff, promotional clutter, and link-shortener noise.

The data has been organized into a strict, scannable architecture:
* `/research/sources.md`: A structured index of all 10 experts with direct links and annotations.
* `/research/linkedin-posts/`: Dedicated text documents housing parsed high-signal strategy data for each specialist.
* `/research/youtube-transcripts/`: Raw JSON transcript blocks pulled directly via API endpoints.
